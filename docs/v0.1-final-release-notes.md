# v0.1 Final Release Notes

发布日期：2026-04-26

## 当前支持能力

- React/Vite/TypeScript 前端 + FastAPI/SQLAlchemy/SQLite 后端。
- AkShare-only 单股票数据层。
- 本地行情/财务/日历缓存与 `data_fetch_log`。
- 研究任务创建与基于真实可得数据的研究报告。
- 数据诊断 API 与报告详情数据诊断展示。
- 观察池与交易监控池。
- Paper Trading only 模拟交易闭环：`Signal -> Risk Check -> Order -> Execution -> Position -> Log`。
- Alembic baseline migration。

## 不支持能力

- 不接真实券商。
- 不接真实 AI。
- 不做真实下单。
- 不做全市场扫描。
- 不做回测。
- 不做新闻公告抓取。
- 不提供投资建议。

## 数据源说明

- 唯一 provider：`AkShare`。
- `provider` 参数为空或 `akshare` 时使用 AkShare。
- 其他 provider 值返回 `UNKNOWN_DATA_PROVIDER`。
- 缓存 TTL：`DATA_CACHE_TTL_MINUTES=1440`。
- 数据刷新失败会保留旧缓存，并在报告/诊断 API 中展示 stale/error。

## 缺失数据表达

- AkShare 未返回财务摘要时，`financialSnapshot=null`。
- 未接新闻/主营构成真实来源时，`newsItems=[]`、`businessSegments=[]`。
- 未接真实 AI 推理时，`aiConfidence=null`。

## Smoke 测试

默认 pytest 使用 AkShare-shaped 本地 fixture，不访问外网。真实环境手动验收：

```bash
python scripts/smoke_akshare_provider.py 600519
python scripts/smoke_real_data_rc.py akshare
```

## 已知限制

- SQLite 仅适合本地演示。
- 无认证、权限、多账户隔离。
- 无异步任务队列。
- AkShare 真实环境结果依赖外部接口稳定性。
- 费用模型为基础模拟，不等同真实券商费率。
