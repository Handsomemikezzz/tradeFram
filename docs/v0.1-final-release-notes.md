# v0.1 Final Release Notes

发布日期：2026-04-26

## 当前支持能力

- React/Vite/TypeScript 前端 + FastAPI/SQLAlchemy/SQLite 后端。
- 统一 API 响应包络。
- 研究任务创建与模板化 mock AI 研究报告。
- MockProvider 默认数据层。
- AkShareProvider 可选真实 A 股基础数据源。
- 本地行情/财务/日历缓存与 `data_fetch_log`。
- 数据诊断 API 与报告详情数据诊断展示。
- 观察池与交易监控池。
- Paper Trading only 模拟交易闭环：
  `Signal → Risk Check → Order → Execution → Position → Log`。
- Alembic baseline migration。

## 不支持能力

- 不接真实券商。
- 不接真实 AI。
- 不做真实下单。
- 不做全市场扫描。
- 不做回测。
- 不做新闻公告抓取。
- 不提供投资建议。

## Paper Trading 规则

- 仅 `PAPER_TRADING_ONLY`。
- 策略信号基于最近 60 日 `price_bar`。
- 默认 MA 参数：
  - `maShortWindow = 5`
  - `maLongWindow = 20`
  - `minBarsRequired = 60`
- A 股模拟交易规则：
  - 买卖数量必须为 100 股整数倍。
  - 买入检查可用现金。
  - 卖出不能超过 `available`。
  - T+1：当日买入不立即进入可卖数量。
  - 模拟成交包含滑点、佣金、印花税。
  - 无行情数据禁止成交。

## 数据源说明

- 默认：`MARKET_DATA_PROVIDER=mock`
- 可选：`MARKET_DATA_PROVIDER=akshare` 且 `AKSHARE_ENABLED=true`
- 缓存 TTL：`DATA_CACHE_TTL_MINUTES=1440`
- 数据刷新失败会保留旧缓存，并在报告/诊断 API 中展示 stale/error。

## AkShare 限制

- 仅支持单只股票查询。
- 依赖外部网络与 AkShare 上游接口。
- 字段缺失时允许降级，并降低 `dataCompleteness`。
- 真实环境验收见 `docs/akshare-validation.md`。

## 风控规则

- `risk_check.status=BLOCKED` 时不得创建 `paper_order`。
- 拒绝场景会写入 `risk_check` 与 `system_log`。
- 默认限制包括：
  - `maxOrderValue`
  - `maxSingleStockPositionValue`
  - `maxTotalPositionRatio`
  - `maxDataStaleMinutes`
  - lot size / fees / slippage

## 交易时间行为

- 本地 demo 默认允许手动巡检：
  - `ALLOW_MANUAL_RUN_OUTSIDE_TRADING_TIME=true`
  - `STRICT_TRADING_TIME_CHECK=false`
- 严格模式：
  - `STRICT_TRADING_TIME_CHECK=true`
  - 非交易时间会被风控拦截。

## 已知限制

- SQLite 仅适合本地演示。
- 无认证、权限、多账户隔离。
- 无异步任务队列。
- AkShare 真实环境结果依赖外部接口稳定性。
- 费用模型为基础模拟，不等同真实券商费率。
