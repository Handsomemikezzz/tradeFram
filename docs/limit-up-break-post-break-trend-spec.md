# 连板断板页 · 断板后走势展开（需求说明）

## 背景与目的

在「连板断板监控」名单中，用户关注的不仅是**哪一天断板**，更是**断板之后股价怎么走**。

典型交易观察逻辑：

- 断板后常会回调 1～2 个交易日；
- 潜在买入观察点出现在**起稳之后**，或**再度出现小幅上涨**时；
- 用户需要自行观察形态，**不需要系统判断「是否企稳」或给出买点建议**。

使用场景包括：

1. **当日**：查看刚进入断板名单的股票，断板后数据可能尚少；
2. **历史回看**：选择约一周前的快照日期，查看当时断板的股票在随后几个交易日内的表现，用于复盘与对比。

本需求是在断板名单表格中，**点击单只股票可展开**，展示该股**断板后的价格变化趋势**。

---

## 功能范围

### 做什么

| 项 | 说明 |
|----|------|
| 交互 | 断板名单每一行可点击展开/收起 |
| 时间锚点 | 以快照中的 **`tradeDate`（断板日）为 T0** |
| 展示区间 | **T0** 起，至 **T+5** 共最多 **6 个交易日**（T0 + 后续 5 个交易日） |
| 数据长度 | **有几天展示几天**，不强制凑满 5 个后续交易日；不足时不补假数据 |
| 价格指标 | **收盘价**（须与断板监控一致：**未复权**日 K） |
| 涨跌展示 | 每个交易日展示 **涨跌幅（%）**，相对**前一交易日收盘价**，用户无需手算 |
| 图表形态 | 迷你**折线图**（纵轴：收盘价；横轴：T0、T+1…）；T0 需可识别为「断板日」 |
| 数据加载 | 建议**懒加载**：仅展开时请求该只股票走势数据 |

### 不做什么（第一版）

- 不判断企稳、不标注买点、不推荐买卖
- 不使用 7 日窗口（只关注断板后约 **1～5 个交易日**）
- 不展示连板阶段的完整 K 线（除非后续单独立项）
- 不把走势数据写入断板快照表（快照仍只存名单字段；走势按需查询日 K）
- 不做盘中实时、分钟线

---

## 涨跌幅口径

- **T0（断板日）**：与名单表格中的 `changePercent` 一致（断板当日相对上一交易日收盘的涨跌幅）。
- **T+1 及以后**：每个交易日相对**前一交易日收盘价**的日涨跌幅。
- 价格与涨跌幅计算须与断板快照生成逻辑一致：**未复权**日 K（`priceAdjustment: none` / `raw`）。

---

## 边界情况

| 情况 | 处理 |
|------|------|
| 当天刚断板，尚无 T+1 | 仅展示 T0；可选提示「后续交易日待更新」，不造假数据 |
| 历史快照（如 5～7 个交易日前） | 通常可展示较完整的 T0～T+5 |
| 停牌断板（`SUSPENDED`） | T0 可能无有效 bar；按实际有数据的交易日展示，并保留断板类型标识 |
| 中间某日停牌 | 该日无数据则跳过或折线中断，不虚构 |
| 「一周」 | 指 **交易日**，非自然日；展示上限为 T+5 共 5 个后续交易日 |

---

## 与现有页面的关系

- **页面**：`连板断板监控`（`LimitUpBreakMonitor`）
- **现有能力**：按日期/阈值查询断板快照名单（代码、名称、断板前高度、当日涨跌幅、成交额、断板类型等）
- **新增能力**：在名单行下展开「断板后走势」区域，不改变快照生成规则与断板判定口径

## 实现参考（非最终实现方案）

### 接口（示意）

```
GET /api/v1/stocks/{code}/post-break-bars
  ?breakDate={快照 tradeDate}
  &maxForwardDays=5
  &adjustment=none
```

响应示例字段：

```json
{
  "breakDate": "2026-05-08",
  "bars": [
    { "tradeDate": "2026-05-08", "close": 12.34, "changePercent": -3.21, "dayOffset": 0 },
    { "tradeDate": "2026-05-09", "close": 12.10, "changePercent": -1.94, "dayOffset": 1 }
  ]
}
```

- `bars`：从 T0 起按交易日顺序排列，至多多 6 条（含 T0）；后续交易日不存在则截断。
- 数据来源：warehouse 未复权日 K（与 `limit_up_breaks` 服务一致）。

### 前端（示意）

- 展开行：`colSpan` 全宽，高度约 160～200px，recharts 折线 + 各点/tooltip 显示涨跌幅。
- 换快照日期或阈值时，建议收起已展开行并清理缓存。

---

## 验收要点

1. 展开后折线以 **T0 收盘价** 为起点，后续点随交易日递增。
2. 每个点可见 **收盘价** 与 **涨跌幅 %**（红涨绿跌，与 A 股习惯一致）。
3. 回看一周前快照时，若日 K 齐全，可看到 **T0～T+5**（共 6 日）；若只有 3 个后续交易日，则只显示 4 个点（含 T0）。
4. 不提供任何「企稳」「建议买入」类文案或标记。
5. 价格口径与断板名单一致（未复权）。

---

## 需求确认记录

- 图表纵轴：**收盘价**（非相对 T0 的百分比坐标）。
- 同时展示：**涨跌幅**，避免用户手算。
- 窗口：**T0 + 最多 5 个后续交易日**；**有几天展示几天**。
- 系统**不**代为判断企稳或买点。
