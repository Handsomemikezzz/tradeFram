# v0.1 RC 验收清单

## MockProvider 验收

- [ ] 默认 `MARKET_DATA_PROVIDER=mock` 可启动后端。
- [ ] 不依赖外部网络即可完成研究与 Paper Trading 闭环。
- [ ] `price_bar` 至少包含 60 条可用于 MA5/MA20 的日线数据。
- [ ] `financial_snapshot`、`market_quote`、`data_fetch_log` 可解释数据来源与刷新状态。
- [ ] `GET /api/v1/data/stocks/{code}/status?provider=mock` 返回完整诊断字段。

## AkShareProvider 验收

- [ ] 仅在 `AKSHARE_ENABLED=true` 时启用。
- [ ] 支持单只股票：`600519`、`600519.SH`、`sh600519` 等输入标准化。
- [ ] 不做全市场扫描、不做新闻公告、不做回测。
- [ ] AkShare 接口失败时不会导致系统崩溃。
- [ ] 失败写入 `data_fetch_log.status=FAILED`，并保留旧缓存。
- [ ] 字段缺失时允许降级，报告展示 `dataCompleteness`。
- [ ] 可用 `scripts/smoke_akshare_provider.py` 或 `scripts/smoke_real_data_rc.py` 手动验证。

## 研究闭环验收

- [ ] 输入股票代码创建研究任务。
- [ ] 研究任务优先读取缓存，缓存过期才刷新 provider。
- [ ] 报告详情展示 provider、更新时间、完整度、缓存命中、过期缓存、最近刷新错误。
- [ ] mock/template AI 只生成研究报告，不直接生成订单。
- [ ] 报告可加入观察池。
- [ ] 报告可加入交易监控池。

## Paper Trading 闭环验收

- [ ] 唯一执行入口为 `POST /api/v1/paper-trading/runs`。
- [ ] 后端内部顺序固定：`Signal → Risk Check → Order → Execution → Position → Log`。
- [ ] Signal Engine 使用最近 60 日 `price_bar` 计算 MA5 / MA20。
- [ ] 风控通过后才创建 `paper_order`。
- [ ] Paper Broker 只产生模拟成交，不接真实券商。
- [ ] Position Manager 更新模拟持仓并关联 `runId` / `traceId`。
- [ ] Trade Logger 写入 `system_log`。

## 数据失败 / 缓存过期 / 数据不完整场景

- [ ] provider 失败写入 `data_fetch_log`。
- [ ] 缓存命中时不重复请求 provider。
- [ ] 缓存过期时尝试刷新。
- [ ] 刷新失败时保留旧缓存，报告提示可能过期。
- [ ] 无日线数据时研究任务失败或降级。
- [ ] 无日线数据时 Paper Trading 不创建成交。
- [ ] `dataCompleteness` 低于 100% 时前端可见。

## A 股模拟交易规则验收

- [ ] 买卖数量必须为 100 股整数倍。
- [ ] 买入前检查可用现金。
- [ ] 卖出不能超过 `available` quantity。
- [ ] T+1：当日买入不立即进入可卖数量。
- [ ] 模拟成交包含佣金、印花税、滑点。
- [ ] 可配置非交易时间禁止成交。
- [ ] 无行情数据禁止成交。
- [ ] 每次风控拒绝必须写 `risk_check` 和 `system_log`。

## 安全边界

- [ ] 仅 `PAPER_TRADING_ONLY`。
- [ ] 不接真实券商。
- [ ] 不接真实 AI。
- [ ] AI 不下单。
- [ ] 前端不能创建成交、持仓或交易日志。
- [ ] `risk_check.status=BLOCKED` 时不得创建 `paper_order`。
- [ ] 所有订单、成交、持仓更新必须关联 `runId` 或 `traceId`。
