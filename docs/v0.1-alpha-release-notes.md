# v0.1 Alpha Release Notes

发布日期：2026-04-26

## 范围

v0.1 Alpha 完成了 A 股研究与模拟交易系统的最小可用闭环：

1. 前端 React/Vite/TypeScript 通过真实 API 调用后端。
2. 后端为 Python + FastAPI + SQLAlchemy + SQLite。
3. 研究任务可生成 mock 研究报告。
4. 报告可加入观察池与交易监控池。
5. 手动运行一次 Paper Trading 检查后，后端按固定顺序执行：
   `Signal → Risk Check → Order → Execution → Position → Log`。
6. 前端可查询订单、持仓、风控检查、系统日志、执行 trace 和 dashboard 摘要。

## 安全边界

- 仅允许 `PAPER_TRADING_ONLY`。
- 不接真实券商。
- 不实现真实自动交易。
- 不接真实 AI。
- AI 只生成研究报告，不直接下单。
- 前端不能直接创建成交、持仓或交易日志。
- `POST /api/v1/paper-trading/runs` 是唯一模拟交易执行入口。
- `risk_check.status = BLOCKED` 时后端不得创建 `paper_order`。

## 已覆盖接口

- P0-A：系统状态、数据源健康、研究任务、研究报告、观察池、监控池、模拟交易引擎、模拟交易运行、账户/持仓、订单、风控、日志。
- P0-B：风险系统状态、最新执行 trace、dashboard overview、monitoring summary、research stats。

## 已覆盖测试

- 非法股票代码。
- 重复加入观察池。
- 重复加入交易监控池。
- 风控 BLOCKED 不创建订单。
- 无监控股票时运行 paper trading。
- 报告不存在返回 404。
- 前端不存在直接调用 `POST /orders`、`POST /executions`、`POST /positions`、`POST /trade-logs` 的逻辑。
- 最小模拟交易闭环。

## 已知限制

- 行情、财务、报告内容均为 seed/mock 数据。
- 策略信号为第一阶段 mock 规则。
- SQLite 仅用于本地开发与演示。
- 未实现认证、权限、多账户隔离、异步任务队列和真实行情刷新。
