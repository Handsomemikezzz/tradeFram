# v0.1 Alpha Acceptance Checklist

日期：2026-04-26

范围：A 股研究与模拟交易系统 v0.1 Alpha。当前版本使用 AkShare-only 单股票数据层，不接真实券商或真实 AI。

## 1. 技术栈验收

- [ ] 后端为 Python + FastAPI。
- [ ] 后端 ORM 为 SQLAlchemy。
- [ ] 后端默认数据库为 SQLite：`sqlite:///./paper_trading.db`。
- [ ] 前端为 React/Vite + TypeScript。
- [ ] 项目没有 Node/Express/NestJS 后端入口。
- [ ] `src/services/api/*.ts` 仅作为 API client、service、类型、adapter、格式化层。

## 2. 研究闭环验收

目标链路：

```text
输入股票代码 → 创建研究任务 → 基于 AkShare 可得数据生成研究报告 → 查询研究记录/报告 → 加入观察池 → 加入交易监控池
```

验收项：

- [ ] `POST /api/v1/research/tasks` 接收 6 位 A 股代码。
- [ ] 非 6 位或非数字股票代码返回统一错误包络，错误码为 `VALIDATION_ERROR`。
- [ ] 不存在于 seed 数据的股票代码返回 `STOCK_NOT_FOUND`。
- [ ] 创建研究任务后返回 `COMPLETED`、`reportId`、`redirectTo`。
- [ ] `GET /api/v1/research/tasks/{taskId}` 可查询任务状态。
- [ ] `GET /api/v1/research/reports/by-code/{code}` 可查询报告详情。
- [ ] 未生成报告的股票查询报告返回 `REPORT_NOT_FOUND`。
- [ ] 研究报告仅整理 AkShare 可得数据，不创建交易订单。
- [ ] `GET /api/v1/research/records` 可查询研究记录。
- [ ] `GET /api/v1/research/stats` 返回研究数量、观察池转化数量、热门行业。
- [ ] `POST /api/v1/watchlist/items` 可加入观察池。
- [ ] 重复加入观察池返回 `WATCHLIST_ALREADY_EXISTS`。
- [ ] `POST /api/v1/monitoring-pool/items` 可加入交易监控池。
- [ ] 重复加入交易监控池返回 `MONITORING_ITEM_ALREADY_EXISTS`。

## 3. 模拟交易闭环验收

唯一执行入口：

```text
POST /api/v1/paper-trading/runs
```

后端内部执行顺序：

```text
Signal Engine → Risk Engine → Order Manager → Paper Broker → Position Manager → Trade Logger
```

验收项：

- [ ] 第一版交易模式始终为 `PAPER_TRADING_ONLY`。
- [ ] 前端只能通过 `POST /api/v1/paper-trading/runs` 触发模拟交易检查。
- [ ] 前端没有 `POST /orders`、`POST /executions`、`POST /positions`、`POST /trade-logs` 调用。
- [ ] `Signal Engine` 仅在后端生成 `signal`。
- [ ] `Risk Engine` 仅在后端生成 `risk_check`。
- [ ] `risk_check.status = BLOCKED` 时不得创建 `paper_order`。
- [ ] `Order Manager` 仅在风控通过后创建 `paper_order`。
- [ ] `Paper Broker` 仅在后端创建 `paper_execution` 并更新订单成交状态。
- [ ] `Position Manager` 仅在后端根据成交更新 `position`。
- [ ] `Trade Logger` 写入 `system_log`，记录 runId/traceId/relId。
- [ ] 所有 order、execution、position 更新都关联 `runId` 或 `traceId`。
- [ ] 无监控股票时运行 `paper-trading/runs` 返回 `COMPLETED`，扫描数为 0，不创建订单。
- [ ] `GET /api/v1/execution-traces/latest` 返回最近一条链路追踪。

## 4. 前端页面验收

- [ ] 顶栏读取 `GET /system/status` 与 `GET /data-sources/health`。
- [ ] Dashboard KPI 读取 `GET /dashboard/overview`。
- [ ] Dashboard 监控摘要读取 `GET /dashboard/monitoring-summary`。
- [ ] 研究页创建任务、任务查询、研究记录、研究统计均读取后端接口。
- [ ] 研究报告详情页读取后端报告，并通过接口加入观察池/监控池。
- [ ] 交易控制台读取监控池、引擎状态、风控状态、最新链路追踪。
- [ ] 交易控制台手动巡检成功后刷新监控池、风控状态、链路追踪。
- [ ] 持仓与日志页读取账户摘要、持仓、订单、风控审计、系统日志。
- [ ] 页面保留 loading、error、empty 状态。
- [ ] 页面/组件直接调用后端 API，不依赖本地静态数据服务。

## 5. 安全边界验收

- [ ] 不接真实券商。
- [ ] 不接真实行情源。
- [ ] 不接真实 AI。
- [ ] 不实现真实自动交易。
- [ ] AI 只生成研究报告，不直接下单。
- [ ] 前端不执行信号生成、风控判断、订单成交、持仓更新、交易日志写入。
- [ ] 风控、订单、成交、持仓、日志均由后端服务层控制。
- [ ] 所有 API 错误使用统一响应包络：`success/error/requestId/serverTime`。
